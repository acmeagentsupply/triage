What triage checks
gateway
Is OpenClaw running and healthy?
sessions
Agent count, orphan detection
disk
Available disk space
verify
Binary checksum vs. release
doctor
openclaw doctor output (25s timeout)
compaction
Context compaction state
activity
Recent event rate across fleet
fleet
Hostname, uptime, identity
Reading the status
[HEALTHY]
All signals clear. System is running normally.
[AT_RISK]
Something is degrading. System still running — catch it before it fails.
[CRITICAL]
Act now. Check signal lines above the status for what failed.
verify states
MATCH
Binary matches release checksum. Trust its output.
MISMATCH
Binary modified since install. Reinstall before trusting output.
UNKNOWN
No authoritative checksum available.
The Proof Bundle  written to ~/octriage-bundles/<timestamp>/
bundle_summary.txt
Version, timestamp, hostname
doctor_output.txt
openclaw doctor output (25s timeout)
gateway_err_tail.txt
Recent gateway error log
gateway_log_tail.txt
Last 120 lines of gateway log
openclaw_status.txt
openclaw status + deep gateway check
verify_integrity.txt
SHA comparison — installed vs. expected
manifest.sha256
Checksums for all bundle files
